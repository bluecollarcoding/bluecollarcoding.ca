• Maintenance & Cleanliness for Repositories
• Documentation Overhaul & Smoke Testing
• Code Linting with Continuous Integration

Software organizations are like car companies; they have engineers to design all the pieces that come together in a finished product. Unlike a car, however, software is constantly being driven around by users and developers. Changing the oil and other little maintenance tasks often fall behind. At Blue Collar Coding, we dig deep into your software project and clean out all the cobwebs. What does this mean?


• improve approachability of repositories to new users
• clarify and document the dev workflow across multiple repos
• find and fix dead links + mark appropriate repos for deprecation
• consolidate duplicate information/files/issues/branches
• review and address stale issues/pull requests
• generally make your organizations' repositories easily navigable

Ah, documentation. Everyone praises it, few write it. By approaching a software project as intentionally naive, we find and fill gaps that might be barriers to entry for student programmers - often this is implicit knowledge or outdated instructions. This can include:


• going through all the docs step by step
• finding and fixing the small gaps
• removing unnecessary complexity
• epxanding the steps that involve implicit knowledge
• consolidation of disparate docs/information
• implementate continuous documentation workflow
• documenting new features (especially if a cli with bash tests)

Every language has a linter yet few projects include it as part of their Continuous Integration from day one. Many of the required lints are often simple but time consuming. We'll tackle them all and get your codebase up to the linting standard, then enforce it by adding the linter to your CI. Linting provides:


• standardized, clean, consistent code
• serendipitious identification of bugs and bad code
• improved documentation
